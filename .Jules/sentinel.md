## 2024-04-18 - Prevent CWE-209 Sensitive Information Exposure
**Vulnerability:** Found multiple instances where FastAPI endpoints were catching generic exceptions and returning their string representation (`detail=str(e)`) in `HTTPException` responses.
**Learning:** This pattern is a CWE-209 (Generation of Error Message Containing Sensitive Information) vulnerability. Exposing internal exception details like stack traces, SQL syntax errors, or internal service states to the client provides attackers with insights into the backend architecture and potential exploitation vectors.
**Prevention:** Always log the full exception securely on the server-side using `logging.error(..., exc_info=True)` for debugging purposes, and return a generic, sanitized message (e.g., "Internal server error") to the client via the HTTP response.
## 2026-04-20 - SSRF Prevention in Nightscout Connection Testing
**Vulnerability:** The `test_nightscout_connection` endpoint parsed user-supplied URLs and made external requests without validating the scheme or resolving the hostname, allowing potential Server-Side Request Forgery (SSRF) against internal network infrastructure.
**Learning:** Backend functions taking arbitrary URLs must validate that the domain resolves to a public, internet-routable IP address before making the request. Trusting the user input blindly can expose internal services, loopbacks, and cloud metadata endpoints.
**Prevention:** Utilize Python's `socket` and `ipaddress` libraries to resolve hostnames and explicitly block private (`is_private`), loopback (`is_loopback`), link-local (`is_link_local`), and multicast (`is_multicast`) address spaces. Furthermore, strictly enforce `http` or `https` URL schemes.
